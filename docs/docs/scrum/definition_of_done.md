# Definition of Done (DoD)

Une User Story est considérée TERMINÉE quand :

## Code ✅
- [ ] Code écrit et fonctionne sans erreur
- [ ] Respect des principes SOLID
- [ ] Design Patterns appliqués si nécessaire
- [ ] Code commenté (JavaDoc)

## Tests ✅
- [ ] Tests unitaires écrits
- [ ] Couverture de tests ≥ 65%
- [ ] Tous les tests passent (mvn test)

## GitHub ✅
- [ ] Code pushé sur GitHub
- [ ] Pull Request créée et mergée
- [ ] Issue GitHub fermée

## Fonctionnalité ✅
- [ ] Démontrée et validée par le Product Owner
- [ ] Critères d'acceptation tous satisfaits
- [ ] Interface utilisateur testée
